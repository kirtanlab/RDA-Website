Road Crossing Assistant

Roads in medium-sized Indian towns often have lots of traffic but no (or disregarded) traffic stops. This makes it hard for the blind to cross roads safely, because vision is crucial to determine when crossing is safe. Automatic and reliable image-based safety classifiers thus have the potential to help the blind to cross Indian roads.

INDRA Dataset

We introduce INDRA (INdian Dataset for RoAd crossing), the first dataset capturing videos of Indian roads from the pedestrian point-of-view.

Dataset Specifications

Our INDRA dataset includes:

  • 104 videos from diverse locations, times, and traffic patterns
  • Collected using an action camera from different roads of Anand, Gujarat
  • Video Format: MOV
  • Resolution: 1920Ã—1080
  • Framerate: 30 fps
  • More than 26,000 frames in total

Annotations

Each video is annotated with:

  • Time instances when the road is safe to cross
  • Time instances when the safe duration ends
  • Binary safety labels for each frame (safe/unsafe)
  • Vehicle bounding box annotations

Model Development

We've experimented with several model architectures to classify road crossing safety, culminating in our novel MobileNetV3-LSTM Hybrid Model that outperforms previous approaches in key metrics.

Model Architectures Explored

1. TimeDistributed CNN-LSTM Hybrid Model

Fig. 1. TimeDistributed CNN-LSTM Hybrid Model

In this baseline model, we used TimeDistributed convolution neural network (CNN) as a feature extractor. The architecture was optimized by experimenting with different numbers of convolution layers and hyperparameters like kernel size, stride, and dilation.

Key features:

  • TimeDistributed wrapper applies convolutional operations independently at each time step
  • Captures temporal patterns and dependencies in the video sequence
  • Extracted features are passed into Time Distributed LSTM layers
  • Input image size of 64 pixels
  • Sigmoid function for binary classification (safe/unsafe)
  • Trained for 100 epochs using binary cross-entropy loss
  • Includes TimeDistributed ReLU and Dropout layers to improve generalization

This architecture uses standard convolution layers, making it suitable for deployment purposes, but it performed poorly on both training and test data.

2. ResNet50-LSTM Hybrid Model

Fig. 2. Residual block of ResNet

Fig. 3. ResNet50-LSTM Hybrid Model

To improve upon the TimeDistributed CNN-LSTM model, we integrated ResNet-50 as a backbone by removing the last two layers from the original architecture.

Key features:

  • Added Region Proposal Network (RPN) inside ResNet layers to suggest potential regions of interest
  • Residual blocks address the vanishing gradient problem through shortcut connections
  • Improved training convergence and model accuracy through residual connections
  • Enhanced feature extraction and representation
  • Better ability to capture long-range dependencies

The architecture was improved by experimenting with different LSTM hidden sizes and layer counts, varying image sizes, and applying normalizing techniques specific to ResNet. We used Adam optimizer with a starting learning rate of 0.001.

3. MobileNetV3-LSTM Hybrid Model (Our Proposed Solution)

Fig. 5. Sigmoid and swish nonlinearities and their "hard" counterparts

Fig. 4. Performance Comparison of Different MobileNet Architectures

Fig. 6. MobileNetV3-LSTM Hybrid Model

Our best-performing model uses MobileNetV3 as a feature extractor, which is memory-efficient and lightweight, making it ideal for deployment on embedded systems like the Nvidia Jetson Nano.

Key features:

  • Utilizes h-swish (hard-swish) activation function for computational efficiency and stability
  • Simpler mathematical operations compared to sigmoid, ideal for resource-constrained environments
  • Non-zero gradient characteristics contribute to stable training
  • Reduced computational cost for better performance on mobile devices
  • Uses ImageNet pre-trained weights as a starting point
  • Two LSTM layers with increased batch size (16) and sequence length (20)
  • Added L2 regularization with lambda value of 0.01

This model was trained for 100 epochs using binary cross-entropy logits as the loss function.

Fig. 9. Comparing Different Models with Metrics vs Value Graph

Our results demonstrate that:

  • MobileNetV3-LSTM achieves the best F1-score (0.8729) and recall (0.881) among all tested models
  • Higher recall means our model identifies almost all safe frames, crucial for ensuring pedestrian safety
  • DilatedRoadCrossNet achieves the highest precision (0.9), but our model offers better overall performance
  • Our model is optimized for deployment on resource-constrained devices like the Nvidia Jetson Nano

The F1 score is the harmonic mean of precision and recall, providing a balanced evaluation of the model's performance. Our proposed MobileNetV3-LSTM hybrid model achieves the highest F1 score, making it the most suitable choice for the road crossing assistant application.

Deployment

In our final working prototype of the Road Crossing Assistant, which can be used by blind people in real-time, we have deployed our single frame CNN model on Nvidia Jetson Nano B01. We converted our pretrained TensorFlow model into a TensorRT model to optimize the processing power and achieve low latency with high throughput. The throughput obtained by TensorRT model is 8 fps as compared to 3 fps by simple TensorFlow model.

Deployment Workflow

The Jetson Nano runs the pre-trained TensorRT model on real-time video stream (obtained from the camera module attached to it) and predicts whether it is safe to cross the road seen in the video stream. It also controls the audio output of headphones to communicate with the user through simple audio commands.

About Us

We are a team of dedicated researchers and engineers focused on developing assistive technologies for the visually impaired in India. Our work on the Road Crossing Assistant aims to address a critical safety challenge faced by blind individuals in medium-sized Indian towns.

Project Impact

Our Road Crossing Assistant system offers several key benefits:

  • Provides real-time auditory feedback about road crossing safety
  • Works with a wearable device based on affordable Nvidia Jetson Nano hardware
  • Specifically designed for Indian traffic conditions where formal traffic controls are often absent
  • Addresses a critical safety need for visually impaired individuals
  • Potential to save lives by preventing dangerous road crossing attempts

Practical Applications

The wearable road crossing assistant:

  • Processes camera input in real-time through our DilatedRoadCrossNet model
  • Provides clear audio feedback indicating when it's safe to cross
  • Runs efficiently on battery power for extended periods
  • Is designed to be unobtrusive and comfortable to wear
